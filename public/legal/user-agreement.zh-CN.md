# Lattice 用户协议与本地调用审计告知

版本：v0.1
生效日期：2026 年 5 月 5 日

本协议适用于 Lattice 桌面端应用及其相关功能。用户安装、启动、访问或继续使用 Lattice，即表示已阅读、理解并同意本协议。若用户不同意本协议，应停止使用 Lattice。

本文件为产品协议草案，不构成法律意见。正式对外发布前，建议由具备相应资质的法律顾问结合业务所在地、目标用户、数据处理方式和第三方服务条款进行审阅。

## 1. 服务说明

Lattice 是面向科研、写作、计算和自动化工作流的桌面端工具。应用可能提供以下能力：

- 管理本地 workspace、文件、项目和运行环境。
- 调用用户配置的模型服务、API 服务、MCP 服务、插件或外部工具。
- 在 Creator、LaTeX、论文写作、代码生成、计算脚本、Agent 工具调用等模块中辅助生成、分析或处理内容。
- 在本地记录部分调用信息，用于调试、问题追踪、性能分析、安全审计和复现排查。

Lattice 本身不保证任何 AI 输出、计算结果、论文内容、代码片段或分析结论的准确性、完整性、适用性或可发表性。用户应自行审查、验证并承担使用结果的责任。

## 2. 账户、配置与本地环境

Lattice 可能允许用户配置第三方模型提供方、API Key、MCP Server、插件、代理、运行命令和本地工作目录。用户应确保其配置来源合法、安全，并有权访问相关服务、数据和文件。

用户应妥善保管自己的设备、系统账户、API Key、访问令牌、工作区文件和网络环境。因用户设备失控、密钥泄露、插件来源不可信、外部服务异常或本地配置不当造成的损失，由用户自行承担。

## 3. AI 与第三方服务

当用户在 Lattice 中调用第三方模型、API、MCP Server、插件或其他外部服务时，相关输入、上下文、文件片段、提示词、工具参数或生成结果可能会被发送至对应服务提供方处理。

用户理解并同意：

- 第三方服务由对应提供方独立运营，Lattice 不控制其数据处理、安全策略、可用性、价格、速率限制或输出质量。
- 用户应自行阅读并遵守第三方服务的服务条款、隐私政策、许可协议和使用限制。
- 用户不得通过 Lattice 向第三方服务提交无权处理的数据、违法内容、侵犯他人权益的材料、受限敏感信息或违反第三方条款的请求。
- 第三方服务产生的费用、配额消耗、封禁风险、服务不可用或结果错误，由用户根据其与第三方服务的关系自行承担。

## 4. Workspace 与本地文件操作

Lattice 的 workspace 功能可能读取、创建、修改、删除、索引或执行用户指定目录中的文件和命令。用户应确认所选 workspace 不包含其无权处理的文件，且相关操作符合组织政策、保密要求和适用法律。

涉及命令执行、脚本运行、依赖安装、文件批量处理、编译、测试或自动化 Agent 操作时，用户应理解这些操作可能改变本地文件、产生新文件、访问网络资源、消耗计算资源或触发第三方服务调用。

## 5. 本地调用审计日志

为提升可观测性、排查问题、分析性能和支持安全审计，Lattice 可能在用户本地设备记录 API 调用和工具调用相关信息。记录范围可能包括：

- 模型调用、流式调用的开始与结束、Provider、Model、状态、耗时、错误摘要和 token 或用量信息。
- workspace IPC、workspace bash、文件操作、计算脚本保存和相关模块调用摘要。
- Creator、LaTeX、论文写作、Agent 工具调用、插件调用、MCP 工具调用等模块的调用事件。
- 请求和响应的结构化摘要，例如输入字段、输出类型、文本长度、二进制大小、哈希摘要、错误类别和调用上下文。
- 与调用相关的时间戳、会话标识、工作区标识、模块名称、操作名称和性能指标。

审计日志默认存储在本地应用数据目录下的 `logs/api-calls/` 子目录中，例如 Electron 的 `userData/logs/api-calls/`。日志采用面向追加写入和批量落盘的方式，并可能使用压缩、轮转和队列机制以提升高并发场景下的写入性能和存储效率。

## 6. 敏感信息处理

Lattice 会尽合理努力减少审计日志中的敏感信息。例如：

- 对 API Key、Token、Authorization、Cookie、Secret、Password 等常见敏感字段进行脱敏或省略。
- 对大型文本、文件内容或二进制内容优先记录长度、类型、摘要或 SHA-256 哈希，而不是完整内容。
- 对错误对象和调用参数进行结构化摘要，避免无必要地保存完整上下文。

但用户应理解，审计日志仍可能包含文件路径、项目名称、提示词片段、工具名称、错误信息、模型名称、调用参数结构、工作流上下文或其他可识别业务内容的信息。用户不应在 Lattice 中处理其无权处理或不希望被本地日志记录的信息，除非已确认相关日志设置符合其需求。

## 7. 用户控制与删除

用户可以根据应用提供的设置、环境变量或本地文件管理能力控制审计日志。例如，当前实现可通过 `LATTICE_API_AUDIT_ENABLED=0` 禁用 API 调用审计记录。后续版本可能在应用设置页中提供更细粒度的日志开关、保留周期、导出和清理能力。

用户可以在本地设备上删除审计日志文件。删除日志可能影响问题排查、错误复现、性能分析和安全审计。若用户所在组织对日志保留、科研数据、保密信息或合规审计有特殊要求，用户应自行确认其使用方式满足相关要求。

## 8. 插件、MCP 与外部工具风险

Lattice 可能支持插件、MCP Server、外部命令或用户自定义工具。这些扩展能力可能读取 workspace 内容、调用网络服务、执行本地命令、生成或修改文件，或访问用户配置的凭据。

用户应仅安装、启用和运行可信来源的插件、MCP Server 和外部工具。因第三方插件、MCP Server、脚本、命令或依赖包造成的数据泄露、文件损坏、服务滥用、费用损失或安全事件，由用户自行承担，除非适用法律另有强制规定。

## 9. 用户禁止行为

用户不得使用 Lattice 从事以下行为：

- 违反适用法律法规、监管要求、科研伦理或组织合规政策。
- 未经授权访问、复制、处理、上传、披露或破坏他人数据、系统、账号或知识产权。
- 提交恶意代码、凭据窃取脚本、攻击载荷、规避安全措施的请求或其他高风险内容。
- 滥用第三方模型、API、插件、MCP Server 或计算资源。
- 将 AI 输出冒充为未经审查的事实、实验结果、审稿意见、法律意见、医疗建议、投资建议或其他专业结论。

## 10. 知识产权与内容责任

用户保留其依法拥有的输入内容、workspace 文件、项目数据和生成成果中的权利。用户应确保其输入、引用、上传、生成、保存和发布的内容不侵犯他人知识产权、商业秘密、隐私权、肖像权、名誉权或其他合法权益。

AI 生成内容可能与已有公开内容相似，也可能包含事实错误、引用错误、许可证不兼容或不适合公开发布的内容。用户在使用、发表、提交、商用或分发前，应自行进行查重、事实核验、引用核验、许可证审查和人工复核。

## 11. 数据安全与备份

用户应自行维护重要文件、项目、论文、实验数据、配置、密钥和日志的备份。Lattice 不承诺对用户本地数据进行云端备份，也不保证任何文件操作、自动化任务、外部命令或第三方服务调用不会造成数据丢失、覆盖、损坏或泄露。

对于重要 workspace，建议用户在启用自动化编辑、批量处理、命令执行、插件调用或 Agent 操作前，使用版本控制、快照、备份目录或其他可靠方式保留可恢复副本。

## 12. 协议更新

Lattice 可能因功能调整、日志能力变化、第三方服务变化、合规要求或安全原因更新本协议。更新后的协议可能通过应用内弹窗、设置页、发布说明、仓库文档或其他合理方式提示用户。

若更新涉及重要权利义务、数据处理范围或审计日志策略的实质变化，Lattice 应尽合理努力提供醒目提示。用户在协议更新后继续使用 Lattice，视为接受更新后的协议。
