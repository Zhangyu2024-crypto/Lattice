  一、目标与参考

  参考 docs/MIGRATION_PLAN.md §4.3，lattice-cli 的 Sidebar 视图是 VSCode
  风格的多视图切换器：Explorer / Library / Knowledge / Compute / Structure
  …。当前 App 的 ActivityBar 只是"打开 Modal"的快捷键，Sidebar.tsx 只渲染固定的
  Session 视图。本次重构把 ActivityBar 改造成真正的视图路由，让 Library 和 
  Compute 成为 Sidebar 内并列的一等视图。

  二、目标架构

  ActivityBar (icons)              Sidebar (activeView 路由)
  ─────────────────                ───────────────────────────
  [Session]  ─────────────────►    SessionView (现有 Sidebar 内容)
  [Library]  ─────────────────►    LibraryView (集合 + 论文列表)
  [Knowledge] ────────────────►    KnowledgeView (图谱入口 + 最近查询)
  [Compute]  ─────────────────►    ComputeView (脚本/片段/健康)
  ...

  - Sidebar 只放"导航 + 列表"；点击条目把"重内容"送到 EditorArea 作为
  artifact/tab：
    - Library：点论文 → 在 EditorArea 开 paper artifact / PDF viewer
    - Compute：点脚本或 "New Script" → 开 compute-pro artifact（复用
  ComputeProWorkbench）
    - Knowledge：点材料 → 开 material-comparison / knowledge-graph artifact
  - 旧的 LibraryModal / KnowledgeBrowserModal
  短期保留作为「全屏浏览」备用入口（命令面板触发），后续再决定是否删除。
